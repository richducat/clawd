# 2026-02-15

## OpenClaw dual-Mac stabilization (office + travel)
- User reported that using travel laptop seemed to override/break office Mac OpenClaw instance.
- Diagnosed that stability requires strict separation of OpenClaw state/identity + only one gateway service per machine.

### Office Mac (home “brain”) — chosen approach
- Use dedicated profile: `openclaw --profile office ...` (state dir: `~/.openclaw-office`).
- Ensure config includes:
  - `gateway.mode=local`
  - `gateway.bind=loopback`
- Ensure only one LaunchAgent exists/loaded:
  - `~/Library/LaunchAgents/ai.openclaw.office.plist`
  - `launchctl list | grep -i openclaw` shows only `ai.openclaw.office`

### Travel Mac (cockpit)
- Use default profile (`~/.openclaw`) or separate `--profile travel`.
- Do NOT copy/sync `~/.openclaw*` between machines.

### Quick checks
- Office: `openclaw --profile office gateway probe`
- Travel: `openclaw gateway probe`
