# TYFYS – 7-Day Fulfillment Throughput + Provider Handoffs (Tasks-only)

## Goal
Devin + Karen can execute fulfillment without Richard’s direct oversight by:
- keeping Deal stages accurate
- using **Provider Assigned** (existing field) as the handoff trigger
- driving every case forward with **Zoho Tasks** (no WhatsApp required)

## Core fields (Zoho Deals)
- Provider Assigned field (existing) → use as handoff flag (Karen owns once assigned)
- Stages used in pipeline:
  - Intake (Document Collection)
  - Ready for Provider
  - Sent to Provider

## Handoff contract
### Devin owns
- Intake (Document Collection)
- moves to Ready for Provider only when provider-ready

### Karen owns
- once Provider Assigned is set (and/or stage is Ready for Provider)
- drives provider handoff + follow-ups

## Daily cadence (Tasks-only)
### 9:00am ET – Devin Triage
- Pull Intake deals + stuck cases
- Create/refresh tasks for each case with due dates
- Promote provider-ready cases to Ready for Provider

### 2:00pm ET – Karen Provider Push
- Pull Ready for Provider + Sent to Provider (no movement in 3 biz days)
- Assign provider (Provider Assigned field)
- Send packet; set ETA; create follow-up tasks

### 4:30pm ET – Devin Close Loops
- Process inbound docs/client replies
- Update stages + tasks so tomorrow’s triage is clean

## 7-day execution
### Day 1: Inventory + stage hygiene
- Ensure every deal has a next action task + due date
- Identify top batch to assign providers

### Day 2: Convert Intake → Ready
- Hard-push doc collection; convert as many as possible

### Day 3: Convert Ready → Sent
- Provider assignment + initial sends

### Day 4: Stall-busters
- Stalled >7d list; each gets one decisive action + follow-up date

### Day 5: QA pass
- Reduce rework: standardize intake package completeness

### Day 6: Weekend cleanup
- Light follow-ups + updates

### Day 7: Weekly reset
- Summarize moves and blocked providers; set next week priorities
